**AliExpress Manual Testing Project
Project Overview**
This project focuses on manual software testing of the AliExpress web application, specifically covering the User Registration, Login, OTP Verification, and Sign Out functionalities.
The goal of this project is to identify functional, validation, UI, and security-related issues through structured manual testing and proper documentation.
This repository contains Excel-based test cases designed following real-world QA practices and industry standards.

**Application Under Test (AUT)**
**Website**: https://www.aliexpress.com/
**Platform**: Web Application
**Testing Type:** Manual Testing

**Scope of Testing**
  -In-Scope Features
  -User Registration
  -Email-based Sign Up
  -Social Sign Up (Google, Facebook)
  -OTP Generation and Verification
  -User Login
  -Social Login
  -Forgot Password
  -User Sign Out
  -Session Handling after Logout
  -Browser Compatibility (Basic)
  -Input Validation
  -UI Behavior of Login & Registration Pages

**Out of Scope**
  -Admin Panel Testing
  -Backend & Database Testing
  -API Testing
  -Performance & Load Testing
  -Security Penetration Testing
  -Automation Testing (Selenium, Cypress, etc.)
  -Payment Gateway Testing
  -Test Artifacts
**The following testing documents are included in this repository:**

Each test case includes the following fields:
  -Test Case ID
  -Module
  -Type of Testing
  -Feature
  -Test Case Description
  -Expected Result
  -Actual Result
  -Test Data
  -Reproducing Steps
  -Bug Screenshot (if any)
  -Developer Comment
  -Final Status
  -Remarks

**Testing Approach**
  -Requirement-based manual testing
  -Positive and negative test scenarios
  -Validation and boundary testing
  -UI and usability testing
  =Basic security checks (session handling)

**Key Findings**
  -Some validation and OTP handling scenarios did not behave as expected
  -Error messages were unclear in certain negative cases
  -Session handling after logout worked correctly
  -Social login functionality performed as expected

**Conclusion**

This manual testing project demonstrates a structured approach to testing a large-scale e-commerce application.
The test cases were designed to simulate real user behavior and identify functional gaps in the authentication flow.
